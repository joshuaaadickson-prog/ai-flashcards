1. Understanding Your AI "Brains"

🌩️ Cloud AI (Gemini Studio)

Runs on Google's servers. Best for scanning complex images, diagrams, and handwriting. Requires an API key and has daily rate limits.

👁️ On-Sight AI (Gemini Nano)

Runs locally in your browser. No API key needed, zero latency, and 100% private. Best for rapid-fire grading and unlimited study sessions.

⚛️ Hybrid Mode (Recommended)

The best of both. Uses Cloud AI to analyze your notes deeply, then uses On-Sight AI to generate and grade your quiz with near-instant speed.

2. The Master Setup Guide

To use On-Sight AI (Gemini Nano), you must manually grant Chrome permission to download the local model. Follow these exact steps:

  1. Stage 1: Grant Permissions

    Type chrome://flags in your address bar.

    • Enable Prompt API for Gemini Nano.
    • Enable Optimization Guide On Device Model (Crucial: Set this to Enabled BypassPerfRequirement).
    • Click Relaunch at the bottom.
  2. Stage 2: Download the Model

    Type chrome://components in your address bar.

    • Find Optimization Guide On Device Model (Note: It is NOT "Optimization Hints").
    • If the version is 0.0.0.0, click Check for Update.
    • Keep Chrome open for 10-15 minutes while it downloads (Approx. 1.5GB).
    • Once the status says "Component updated", your brain is installed!
  3. Stage 3: Verification

    Open the console (Right Click > Inspect > Console) and type:

    await (await ai.languageModel.capabilities()).available;

    If it returns "readily", you are connected!

3. Critical Troubleshooting (Read if stuck)

  • Optimization Guide is COMPLETELY MISSING?

    If you don't see any "Optimization Guide" entry in chrome://components even after Stage 1, follow the Nuclear Checklist:

    • The Flag Choice: Go back to chrome://flags. Many people just select "Enabled". You MUST select "Enabled BypassPerfRequirement". Relaunch twice.
    • What about "Enabled Multilingually"? This option is for the Prompt API flag. While good for language support, it DOES NOT count as bypassing performance requirements. You still need the BypassPerfRequirement setting on the Optimization Guide flag to trigger the actual download.
    • The Hard Restart: Type chrome://restart in your address bar. This forces a hard kill and restart of every process that "Relaunch" might miss.
    • The Chrome Channel: If it's still missing, you may need Chrome Canary or Chrome Dev. Stable Chrome (127+) supports it, but the component might not register on all hardware configurations without the Canary build.
  • Advanced Diagnostic: The "Internal Monitor"

    Type chrome://on-device-internals in your address bar to see exactly what Chrome thinks of your hardware diagnostics.

    • Check Criteria: In the "Foundational Model" tab, ensure Device Capable and Disk Space Available are both true.
    • The Toggle: Look for kPromptApi in the feature list adaptations. Click the "set to true" button. This forces Chrome to acknowledge that a feature is requesting the model.
    • The Console Signal: Open this app's console (Inspect > Console) and type: await ai.languageModel.create();. It may error, but it sends the necessary "wakeup" signal.
    • Verify: Go back to chrome://components and check if Optimization Guide has appeared or started an update.
  • Stuck at "Finalizing Deck"?

    Generating 20 high-quality cards is a heavy task. The Finalizing phase typically takes an extra 15-30 seconds to format and verify the deck. Please be patient while the progress bar sits at 0.

  • Silent Download Failure?

    Ensure you have at least 22GB of free disk space. Chrome will stop the download without warning if space is low.

  • The Force Method:

    If the component still won't update, go to chrome://on-device-internals and click Force Download on the Model Status tab.
